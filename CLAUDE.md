## Project Overview

ERGMRank.jl is a Julia port of the R `ergm.rank` package (from the StatNet collection) for fitting Exponential Random Graph Models to rank-order networks, where each actor assigns ordinal ranks to a subset of other actors.

## Development Commands

- **Run tests:** `julia --project=. -e 'using Pkg; Pkg.test()'`
- **Build docs:** `julia --project=docs docs/make.jl`
- **Load package in REPL:** `julia --project=.` then `using ERGMRank`

## Architecture

The entire package lives in a single source file: `src/ERGMRank.jl`. It is organized into these sections:

1. **RankNetwork** -- dense `Matrix{Int}` of rank values; row i = ego i's ranking of the alters, GREATER value = HIGHER standing (ergm.rank convention). The complete-ranking invariant (each row's off-diagonal is a permutation of `1:(n-1)`) is validated at construction, preserved by `swap_ranks!` (the AlterSwap move), and checkable with `is_valid_ranking`.
2. **Reference measure** -- `CompleteOrderReference`: discrete-uniform over complete orderings (constant; cancels from likelihood ratios; the constraint is what matters).
3. **Terms** -- `RankDeference`, `RankNonconformity(:all/:localAND)`, `RankNodeICov`, `RankInconsistency`, `RankEdgeCov`, implemented from ergm.rank's `wtchangestats_rank.c` summary functions and golden-master tested against R ergm.rank 4.1.2 (values in test/runtests.jl).
4. **Estimation** -- `ergm_rank` (alias `fit_rank_ergm`): swap-based maximum pseudo-likelihood — for each ego and unordered alter pair, the observed relative order is logistic in θ'[g(y) − g(y_swapped)]; maximized with the shared `ERGM.newton_fit` (Newton-Raphson with step-halving). **Claim discipline:** this is NOT the MCMC MLE and must not be described as a "consistent" approximation to it — the swap comparisons multiplied into the pseudo-likelihood are not independent, and no asymptotic regime under which swap-MPLE is consistent has been established here. The DEFAULT SEs (`se=:hessian`) are the inverse observed pseudo-Hessian and are therefore expected to be anticonservative under dependence; say so wherever the estimator is documented.

   **Robust SEs** (`se_type` on `RankERGMResult`, reported by `Networks.se_method`): `se=:bootstrap` is a **parametric bootstrap** — simulate `n_boot` rank networks at θ̂ with the AlterSwap sampler (`simulate_rank_ergm`), refit the swap MPLE on each (`_rank_mple_fit` is the shared core), empirical covariance — with the same API as `ERGM.mple`'s (`n_boot`/`boot_burnin`/`boot_interval`/`rng`), running on the ONE shared `Networks.bootstrap_cov` loop; never paste the loop in. The point estimates are identical under both; only the covariance changes. On the test fixture the bootstrap SEs are more than **2× larger** than the Hessian ones — this package is where the anticonservatism bites hardest, because *every* rank fit has overlapping comparisons and there is no exact special case to exempt. `show`/`approximations` read `se_type` and drop the anticonservatism claim when a bootstrap was used, but keep the point-estimate caveat (swap pseudo-likelihood, no consistency claimed) in both — `is_exact` stays unconditionally `false`.
5. **Simulation** -- `simulate_rank_ergm`: Metropolis with the symmetric AlterSwap proposal; every state is a valid complete ranking.

## The swap MPLE is a logistic regression, and its loop is ERGM.jl's (review finding 15)

The swap pseudo-likelihood **is** a logistic likelihood on the swap-difference rows `d = g(y) − g(y_swapped)` with the response identically `true` — the observed order is always the "success". So `_rank_mple_fit` no longer carries its own loop (with `hess .-= (pr*(1-pr)) .* (d * d')`, a fresh `p×p` outer product on every one of the `n(n−1)(n−2)/2` comparisons of every Newton evaluation: **229 KB and 0.20 ms per evaluation** on a 17-actor ranking). It builds the design with `_rank_design(terms, work)` — one dense `(comparisons × p)` matrix, not a vector-of-vectors — and hands it to the shared **`ERGM.logistic_derivatives(D, trues(size(D, 1)))`**, the same builder ERGMMulti's MPLE and TERGM's CMPLE run on: **192 bytes and 0.046 ms per evaluation**, 4.3x faster, allocations independent of the comparison count. **Never paste the loop back in**; if it needs a feature, add it to `ERGM.logistic_derivatives`. An `@allocated` regression test pins the bound, and a companion test asserts `logistic_derivatives(D, trues(...))(θ̂)` reproduces `fit.loglik` — i.e. that the all-true logistic really *is* the estimator, not merely similar to it.

## Golden fixtures (statnet ergm.rank)

`test/fixtures/newcomb_rank.toml`, generated by `test/fixtures/r/newcomb_rank.R` (**slow — ~17 min**: a rank-ERGM MCMLE takes ~3 minutes and the script does six, for the replication study) and loaded with Networks.jl's `load_golden`. Model: `newcomb[[1]] ~ rank.deference + rank.nonconformity("all")`, `reference = ~CompleteOrder`.

**The two packages do not fit the same estimator, and the fixture measures the gap rather than hiding it.** `ergm.rank` fits the MCMC MLE; `ergm_rank` fits the swap pseudo-likelihood. A pseudo-likelihood is not a noisy MLE — it is a different estimator — so the coefficient comparison is **`@test_broken`**, and what the testset *asserts* is the **character** of the disagreement:

- **The observed sufficient statistics match exactly** (844, 12748) — deterministic, asserted at 1e-9. The term formulas are right.
- **The gap is systematic, not Monte-Carlo.** swap-MPLE `[-0.14091, -0.0058538]` vs ergm.rank `[-0.15310, -0.0065927]`. ergm.rank's own seed-to-seed sd is 7.4e-4 / 5.6e-5; the gap is **16x and 13x** that. No MCMC budget closes it.
- **But it is small where it counts:** 0.30 and 0.43 of an ergm.rank standard error. Same sign, same order, same substantive story. A different estimator, not a broken one — which is exactly what this package claims.
- **The pseudo-Hessian SEs are anticonservative, and now by a measured factor:** 3.9x (deference) and 2.0x (nonconformity) narrower than the MLE's. `se=:bootstrap` moves most of the way back (0.0280 vs R's 0.0404), and the testset asserts it does.

Do **not** widen a tolerance to turn the `@test_broken` green. When a real MCMLE lands, delete it and assert against `mle_coefficients` at a multiple of the frozen `mle_seed_sd`, as the ERGM.jl fixture does.

## Key Dependencies

- **ERGM.jl** -- Provides the `AbstractERGMTerm` base type and the `name`/`compute` generics (extended via `import ERGM: name, compute`)

Requires Julia 1.12+.

## Conventions

- All term structs subtype `AbstractERGMTerm` and implement `name()` (returning a dotted string like `"rank.mutual.3"`) and `compute()` (returning `Float64`).
- `RankNetwork` stores a dense `Matrix{Int}`; it does not wrap a `Network`.
- Rankings use values `1:(n-1)` per ego where GREATER value = HIGHER standing (matches R ergm.rank; note this is the opposite of "rank 1 = best").
- Functions mutating a `RankNetwork` use the `!` suffix convention (`set_rank!`).
- The module exports all public API symbols explicitly at the top of the module definition.
